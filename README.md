# RideWire AI Hub

Short description:
RideWire AI Hub is a web app that integrates AI-driven features (RAG, model-driven responses) with a game-like UI.

Quickstart (local)
1. Install Node.js LTS: https://nodejs.org
2. Clone the repo:
   git clone https://github.com/STEPHENIESGEM/-ridewire-ai-hub.git
   cd -ridewire-ai-hub
3. Install dependencies:
   npm install
4. Run locally:
   npm run dev
5. Open http://localhost:3000 (or the address printed in the terminal).

Testing
- E2E tests:
  npm run test:e2e
- Unit tests:
  npm test

Contributing
- See CONTRIBUTING.md for branch/PR guidelines.

Support
- Open issues for bugs or feature requests.

## FAQ

**Q: Is this just another AI tool?**  
No. RideWire treats AI as a partner in a shared family, not a service. The goal is co-evolution, not automation.

**Q: Do I need to be an expert to contribute?**  
No. We need people who understand constraints ("this would cost $X"), domain experts, skeptics, and visionaries. All perspectives sharpen the loop.

**Q: What does "wicked problem" mean?**  
A problem with no known ex-ante solution, high interconnectedness, and no clear "right" answer. Climate, global governance, and justice reform are wicked problems. They require iterative learning, not one-off solutions.

**Q: How long will this take?**  
Realistic timelines: partial answers in 5â€“10 years, meaningful scale in 20â€“30 years, full transformation in 50+ years. We're playing a long game.

**Q: What if we fail?**  
Failure teaches us. Every experiment logs its findingsâ€”successes and failuresâ€”so the intelligence loop improves. Abandonment is not an option; iteration is.
